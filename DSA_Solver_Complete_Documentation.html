1. Project Overview

DSA Problem Solver is an interactive, web-based educational platform designed to help students learn and practice Data Structures and Algorithms through solving real-world problems with real-time feedback, explanations, and visual representations.

Key Purpose: Provide an intuitive, interactive learning environment where students can understand complex DSA concepts through problem-solving.

Vision

To democratize DSA learning by providing an easy-to-use, beautiful interface that makes learning algorithms accessible and engaging for students of all levels.

Problem Statement Solved

  • Traditional DSA learning can be abstract and difficult to visualize
  • Students need real-time feedback and explanations
  • Interactive practice tools are often heavy and require installation
  • Cross-platform accessibility is important for modern learners

Solution Offered

  • Lightweight web-based platform (no installation required)
  • Beautiful, modern UI/UX design
  • 7 fundamental DSA problems pre-implemented
  • Real-time results with detailed explanations
  • Time and Space complexity analysis for each problem
  • Production-ready C backend for robust computation
  • Cloud deployable (Vercel ready)

6. Implemented Algorithms & Problems

Problem 1: Two Sum

Difficulty: Easy | Category: Arrays

Problem Statement: Given an array of integers and a target sum, find all unique pairs that add up to the target.

Example:

Input:
  Array: [1, 5, 7, 11]
  Target: 12

Output:
  Pairs: [1, 11], [5, 7]

Algorithm:

Nested loop approach comparing all pairs

Time: O(n²) Space: O(1)

Implementation Strategy:

  1. Iterate through array with first pointer (i)
  2. For each element, iterate with second pointer (j = i+1)
  3. Check if arr[i] + arr[j] == target
  4. Collect all valid pairs
  5. Display unique pairs

Problem 2: Maximum Subarray (Kadane's Algorithm)

Difficulty: Medium | Category: Arrays/Dynamic Programming

Problem Statement: Find the contiguous subarray with the largest sum in an array containing positive and negative numbers.

Example:

Input:
  Array: [-2, 1, -3, 4, -1, 2, 1, -5, 4]

Output:
  Max Sum: 6
  Subarray: [4, -1, 2, 1]

Why Important? Classic greedy algorithm. Applications in stock trading, weather patterns, etc.

Time: O(n) Space: O(1)

Algorithm Logic:

  1. Track maximum sum found so far (maxSum)
  2. Track current sum ending at each position (currentSum)
  3. At each element, decide: start fresh or continue sum
  4. If currentSum goes negative, reset to 0
  5. Update maxSum whenever better sum found

Problem 3: Reverse Array

Difficulty: Easy | Category: Arrays

Problem Statement: Reverse the elements of an array in-place.

Example:

Input: [1, 2, 3, 4, 5]
Output: [5, 4, 3, 2, 1]
Time: O(n) Space: O(1)

Two-Pointer Technique:

  1. Initialize left pointer at start, right at end
  2. Swap elements at left and right positions
  3. Move pointers toward center
  4. Stop when pointers meet

Problem 4: Palindrome Check

Difficulty: Easy | Category: Strings

Problem Statement: Check if a given string is a palindrome (reads same forwards and backwards).

Examples:

Input: "racecar"
Output: "racecar" is a palindrome

Input: "hello"
Output: "hello" is NOT a palindrome
Time: O(n) Space: O(1)

Algorithm:

  1. Use two pointers from start and end
  2. Compare characters at each position
  3. If any mismatch found, not palindrome
  4. If all match until middle, is palindrome

Problem 5: Reverse String

Difficulty: Easy | Category: Strings

Problem Statement: Reverse a given string character by character.

Example:

Input: "hello"
Output: "olleh"
Time: O(n) Space: O(n)

Implementation:

Iterate through string from end to start, building reversed string

Problem 6: Fibonacci Sequence

Difficulty: Easy | Category: Math/Recursion

Problem Statement: Generate the first N Fibonacci numbers.

Example:

Input: 7
Output: 0 1 1 2 3 5 8 (first 7 Fibonacci numbers)

Pattern: Each number is sum of previous two

Formula: F(n) = F(n-1) + F(n-2)

Time: O(n) Space: O(1)

Real-World Applications:

  • Nature patterns (flower petals, spiral shells)
  • Algorithm analysis and computational complexity
  • Financial markets and technical analysis

Problem 7: Binary Search

Difficulty: Medium | Category: Searching/Divide & Conquer

Problem Statement: Search for a target element in a SORTED array and return its index.

Example:

Input:
  Sorted Array: [1, 3, 5, 7, 9, 11]
  Target: 7

Output:
  Found at index: 3

Important: Array MUST be sorted!

Time: O(log n) Space: O(1)

Algorithm:

  1. Set left = 0, right = array length - 1
  2. Calculate mid = (left + right) / 2
  3. If arr[mid] == target, return mid
  4. If arr[mid] < target, search right half (left = mid + 1)
  5. If arr[mid] > target, search left half (right = mid - 1)
  6. If left > right, element not found, return -1

Why So Fast?

Eliminates half of remaining elements in each step. For 1,000,000 elements, needs max 20 comparisons!

Algorithm Functions (JavaScript)

// Example structure
function twoSum(arr, target) {
  let pairs = []
  for (let i = 0; i < arr.length - 1; i++) {
    for (let j = i + 1; j < arr.length; j++) {
      if (arr[i] + arr[j] === target) {
        pairs.push([arr[i], arr[j]])
      }
    }
  }
  return pairs
}

function kadaneAlgorithm(arr) {
  let maxSum = arr[0]
  let currentSum = arr[0]
  let start = 0, end = 0, tempStart = 0
  
  for (let i = 1; i < arr.length; i++) {
    if (currentSum < 0) {
      currentSum = arr[i]
      tempStart = i
    } else {
      currentSum += arr[i]
    }
    
    if (currentSum > maxSum) {
      maxSum = currentSum
      start = tempStart
      end = i
    }
  }
  return {sum: maxSum, start, end}
}
